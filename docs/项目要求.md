# AutoX 项目核心要求 (Project Requirements)

本文件定义了 AutoX 项目的所有核心设计原则和技术要求。**后续的所有开发任务、代码修改及架构调整都必须严格遵守本文件中的规定。**

---

## 1. 核心原则：自包含 (Self-Contained)
项目最终目标是构建一个**自包含、零环境依赖**的程序。

### 1.1 环境依赖要求
- **用户端零安装**：用户在使用本程序时，除了显卡驱动外，不应被要求安装任何额外环境（如 Python, CUDA Toolkit, cuDNN, Git, Conda 等）。
- **绿色运行**：程序应支持解压即用，不修改用户系统全局环境变量。

### 1.2 模型与资源管理
- **本地化优先**：所有模型文件（.pt, .engine, .onnx 等）必须存放在程序根目录下的 `models/` 文件夹中。
- **禁止自动下载**：程序严禁在运行时自动从互联网下载核心组件或模型权重。所有必要资源必须在分发包中预置。
- **配置隔离**：禁用第三方库（如 Ultralytics）的全局设置同步功能，确保所有配置均保存在程序目录下。

---

## 2. 性能要求 (Performance)
- **极致低延迟**：推理与采集链路的总延迟必须控制在可接受范围内（目标 < 10ms）。
- **资源占用优化**：在保证性能的前提下，尽可能减少 CPU 和显存的占用，避免影响游戏运行。

---

## 3. 兼容性要求 (Compatibility)
- **系统支持**：完美支持 Windows 10/11 64位系统。
- **硬件降级支持 (TODO)**：
    - 当前阶段：优先确保 NVIDIA 显卡 + CUDA 加速的极致体验。
    - 后续阶段：待核心功能稳定后，考虑支持自动回退到 CPU 推理（通过 OpenVINO 或 ONNX Runtime 优化），以及支持 AMD/Intel 显卡加速。

---

## 4. 开发规范 (Development Standards)
- **模块化设计**：采集、推理、控制、输入模拟必须保持解耦，通过抽象基类（Abstract Classes）进行定义。
- **安全加固兼容性**：代码逻辑必须考虑到后续的 PyArmor 混淆和 Nuitka 编译需求，避免使用动态反射等可能导致编译失败的特性。
- **文档同步**：任何重大架构调整或新增核心功能，必须同步更新 `docs/` 目录下的相关文档。

---

## 5. 变更流程
- 任何违反上述原则（特别是“自包含”原则）的改动，必须在执行前向用户明确说明理由，并提供优化或折中方案。
